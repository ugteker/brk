# Product

## Register

product

## Users

Finance professionals, researchers, and avid content consumers who subscribe to podcasts, newsletters, and news sources. They use the platform while monitoring markets or preparing for decisions — they want signal extracted from noise, not a summary of what they already read.

## Product Purpose

An AI-powered research intelligence platform. Users primarily create AI agents through a curator that turns a freeform conversation into an agent profile; manual setup remains available. The curator collects the intended profile conversationally, then presents an editable review that the user must explicitly confirm before the agent is created. Existing agents can also be refined with **Improve with AI**. Agents have distinct character personas (finance expert, teacher, philosopher, summarizer, etc.), periodically analyze content sources, and produce structured reports. Reports surface in a personal feed, allowing users to triage and act on insights, recommendations, and risks across their subscribed sources.

Guided setup begins with a source. The selected source is passed to the curator as advisory context, not as a binding configuration. Choosing **Set up later** saves that source only; it does not create an agent, playbook, or run. Agent avatars are not currently available, and advanced runtime configuration is outside this flow's scope.

## Brand Personality

Precise, intelligent, ambient. The platform works in the background and delivers quality over volume. Three words: **sharp, focused, trustworthy**.

## Anti-references

- Generic SaaS dashboards with hero-metric cards (big number + label + gradient)
- Cluttered fintech dashboards with every data point competing for attention
- Consumer news-aggregators (e.g. Flipboard) — too editorial, not enough signal precision
- Crypto dashboard aesthetics — too aggressive, loud, gamified

## Design Principles

1. **Signal over noise.** Every UI element earns its place by helping the user process faster, not by adding visual interest.
2. **Persona-aware identity.** Character type is a first-class signal — finance expert looks different from philosopher — so the user knows who produced the insight before they read a word.
3. **Progressive disclosure.** The feed is scannable; the full report is one tap away. Cards never try to show everything.
4. **Emphasis is semantic.** Accent color encodes what kind of report this is (critical risk vs. positive opportunity vs. open question), not the brand's preference for decoration.
5. **Committed to craft.** Dark mode is native, not an afterthought. Spacing, contrast, and motion are deliberate.

## Accessibility & Inclusion

WCAG AA minimum. Dark mode first-class (the primary use context is an evening/night desk). Reduced motion respected. Touch targets adequate for tablet use.
